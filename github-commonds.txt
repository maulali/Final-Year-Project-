git init: Initializes a new Git repository in the current directory. This prepares the folder to start tracking changes.


git commit -m "first commit": Commits the staged files to the repository with the message "first commit". The message is a description of what was changed or added in this commit.

git branch -M main: Renames the default branch to main. By default, Git might create a branch called master, but this command changes it to main.

git remote add origin https://github.com/akarshvyasji/youtube: Adds a remote repository named origin with the specified URL. This URL points to a GitHub repository where the code will be pushed.

git push -u origin main: Pushes the local main branch to the origin repository on GitHub. The -u flag sets the upstream branch, so future git push commands will know to push to origin main by default.

After running these commands, the code and commits will be pushed to GitHub, and any new changes can be updated with a simple git push.

Ye commands ek nayi Git repository ko initialize karne, pehla commit karne, branch set karne, GitHub se connect karne aur code push karne ke liye hain. Yahaan har command ka matlab diya gaya hai:

git init: Yeh command current directory mein ek nayi Git repository initialize karti hai. Yeh folder ko changes track karne ke liye tayar karti hai.

git add README.md: Is command se README.md file ko staging area mein add kiya jata hai, yaani yeh commit ke liye ready hai. Aap kisi specific file ka naam likh sakte hain, ya phir sabhi files ko stage karne ke liye git add . ka use kar sakte hain.

git commit -m "first commit": Yeh command staged files ko commit karti hai aur "first commit" ka message add karti hai. Yeh message batata hai ki is commit mein kya changes kiye gaye hain.

git branch -M main: Yeh command default branch ka naam main rakhta hai. Git mein default branch master ho sakti hai, par yeh command usse main mein change kar deti hai.

git remote add origin https://github.com/akarshvyasji/youtube: Is command se ek remote repository add ki jati hai jise origin kaha jata hai aur uska URL diya jata hai. Yeh URL GitHub repository ka hai jahan aap apna code push karenge.

git push -u origin main: Yeh command local main branch ko GitHub ke origin repository par push karti hai. -u flag se upstream branch set ho jati hai, jisse aage se bas git push command chalakar changes ko push kar sakenge.
